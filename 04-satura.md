1. Software Engineering
Pengertian: Software engineering adalah proses pengembangan perangkat lunak yang dilakukan secara sistematis, terstruktur, dan terencana.
Penjelasan: Dalam pembuatan software, tidak cukup hanya bisa coding, tetapi juga harus mengikuti tahapan dan metode tertentu agar hasilnya berkualitas. Software engineering membantu memastikan bahwa sistem yang dibuat dapat berjalan dengan baik, mudah digunakan, mudah dikembangkan, serta sesuai dengan kebutuhan pengguna dan tujuan bisnis. Dengan pendekatan ini, kesalahan dapat diminimalkan dan proses pengembangan menjadi lebih efisien.

2. SDLC (Software Development Life Cycle)
Tahapan:
Planning (Perencanaan)
Design (Perancangan)
Development (Pengembangan/Coding)
Testing (Pengujian)
Maintenance (Perawatan)
Penjelasan: SDLC adalah siklus hidup dalam pengembangan software yang digunakan sebagai panduan agar proses berjalan terarah. Pada tahap planning, dilakukan analisis kebutuhan sistem. Pada tahap design, dibuat rancangan sistem seperti arsitektur dan tampilan. Tahap development adalah proses pembuatan program. Setelah itu dilakukan testing untuk memastikan tidak ada error. Terakhir, maintenance dilakukan untuk memperbaiki bug dan meningkatkan sistem setelah digunakan. Dengan SDLC, proyek menjadi lebih terstruktur dan mudah dikontrol.

3. Metode Pengembangan
Waterfall
Proses berurutan (linear)
Setiap tahap harus selesai sebelum lanjut ke tahap berikutnya
Penjelasan: Metode Waterfall cocok digunakan jika kebutuhan sistem sudah jelas sejak awal dan jarang berubah. Kelebihannya adalah mudah dipahami dan terdokumentasi dengan baik. Namun, kekurangannya adalah sulit melakukan perubahan jika sudah masuk ke tahap berikutnya.
Agile (Scrum, XP)
Fleksibel dan iteratif
Menggunakan sprint (periode kerja singkat)
Penjelasan: Agile lebih cocok untuk proyek yang dinamis dan sering mengalami perubahan kebutuhan. Pengembangan dilakukan secara bertahap, sehingga hasil dapat dievaluasi secara berkala. Metode ini memungkinkan tim untuk beradaptasi dengan cepat terhadap perubahan dan meningkatkan kualitas produk secara terus-menerus.

4. Jenis Testing
Blackbox Testing
Fokus pada input dan output
Penjelasan: Pengujian dilakukan tanpa melihat kode program. Tujuannya untuk memastikan bahwa fungsi sistem berjalan sesuai dengan kebutuhan pengguna.
Whitebox Testing
Fokus pada struktur dan logika kode
Penjelasan: Pengujian dilakukan dengan melihat bagian dalam program, seperti alur logika, kondisi, dan struktur kode untuk memastikan tidak ada kesalahan dalam implementasi.
Greybox Testing
Gabungan blackbox dan whitebox
Penjelasan: Pengujian dilakukan dengan memahami sebagian struktur sistem, sehingga dapat menguji dari sisi fungsi sekaligus logika secara lebih efektif.

5. Teknologi Pendukung
API (Application Programming Interface)
Penghubung antar sistem
Penjelasan: API memungkinkan dua aplikasi atau lebih untuk saling berkomunikasi dan bertukar data secara otomatis.
JSON & XML
Format pertukaran data
Penjelasan: JSON dan XML digunakan sebagai format standar dalam pengiriman data antar sistem karena mudah dibaca dan diproses.
CMS (Content Management System)
Sistem pengelolaan konten
Penjelasan: CMS memudahkan pengguna dalam mengelola konten website tanpa harus memahami coding, sehingga lebih efisien dan praktis.

6. Peran dalam Tim
Project Manager (PM) → Mengatur jalannya proyek
Developer → Membuat dan mengembangkan sistem
QA (Quality Assurance) → Menguji kualitas software
Designer → Mendesain tampilan (UI/UX)
Analyst → Menganalisis kebutuhan sistem
Penjelasan: Setiap anggota tim memiliki peran dan tanggung jawab masing-masing. Kolaborasi yang baik antar tim sangat penting agar proyek dapat berjalan lancar, selesai tepat waktu, dan menghasilkan software yang berkualitas.

7. Estimasi Proyek
COCOMO (Constructive Cost Model)
Digunakan untuk memperkirakan biaya dan waktu
Penjelasan: COCOMO membantu dalam menentukan jumlah tenaga kerja, waktu pengerjaan, dan biaya yang dibutuhkan berdasarkan kompleksitas proyek.
SLOC (Source Lines of Code)
Mengukur jumlah baris kode
Penjelasan: SLOC digunakan untuk memperkirakan ukuran software, yang kemudian dapat digunakan sebagai dasar dalam perhitungan estimasi proyek.

8. Inti Materi
Penjelasan:
Secara keseluruhan, materi ini menjelaskan bagaimana proses pengembangan software dilakukan secara sistematis mulai dari perencanaan hingga perawatan. Dengan menggunakan metode yang tepat, pengujian yang baik, teknologi pendukung, serta kerja sama tim yang efektif, sebuah software dapat dikembangkan dengan kualitas yang baik, efisien, dan sesuai dengan kebutuhan pengguna.
