ANALISIS FRAMEWORK PREP
framework PREP (Point–Reason–Example–Point). Ini biasanya dipakai buat bikin argumen yang jelas dan kuat.
Point (Pernyataan Utama)
Proyek Content Management System (CMS) layak untuk dikerjakan karena mampu meningkatkan efisiensi pengelolaan konten dan kualitas layanan sistem secara signifikan.

Reason (Alasan)
CMS memungkinkan pengguna (admin/non-programmer) untuk membuat, mengedit, dan mengelola konten tanpa harus memahami coding. Selain itu, CMS juga menyediakan fitur seperti manajemen user, keamanan, dan struktur data yang lebih rapi sehingga proses kerja menjadi lebih cepat, aman, dan terorganisir.

Example (Contoh)
Misalnya pada website sekolah:
Tanpa CMS → update berita harus lewat developer (lama & ribet)
Dengan CMS → admin sekolah bisa langsung upload berita, foto, atau pengumuman lewat dashboard

Hasilnya:
Informasi lebih cepat sampai ke siswa
Tidak tergantung pada programmer
Lebih hemat biaya operasional

Point (Penegasan Ulang)
Dengan demikian, proyek CMS layak untuk dikerjakan karena memberikan kemudahan, efisiensi, serta peningkatan kualitas pengelolaan informasi dalam sebuah sistem.

ANALISIS FRAMEWORK PIECES
dalam Rekayasa Perangkat Lunak, dan framework PIECES memang sering dipakai buat nentuin apakah suatu proyek itu go / no-go.
Apa itu PIECES?
PIECES adalah metode analisis untuk mengevaluasi sistem dari 6 aspek:
P → Performance (Kinerja)
I → Information (Informasi)
E → Economy (Ekonomi/Biaya)
C → Control (Kontrol/Keamanan)
E → Efficiency (Efisiensi)
S → Service (Layanan)

Kenapa proyek CMS layak (GO) pakai PIECES?
CMS (Content Management System) itu cocok banget dianalisis pakai PIECES karena sistemnya berhubungan langsung dengan pengelolaan konten, 
user, dan performa website. Ini penjelasan tiap aspek:
1. Performance (Kinerja)
CMS bisa meningkatkan kecepatan update konten tanpa coding manual
Admin bisa langsung publish artikel → lebih cepat dari sistem lama
Kesimpulan: Lebih cepat → layak (GO)

2. Information (Informasi)
Informasi lebih terstruktur (artikel, kategori, user)
Data lebih mudah dicari & dikelola
Kesimpulan: Kualitas informasi meningkat → GO

3. Economy (Ekonomi)
Mengurangi biaya developer untuk update konten
Bisa pakai CMS open-source (hemat biaya)
Kesimpulan: Lebih hemat → GO

4. Control (Kontrol)
Ada login, role (admin, editor, user)
Keamanan lebih terkontrol dibanding sistem manual
Kesimpulan: Lebih aman → GO

5. Efficiency (Efisiensi)
Tidak perlu coding tiap update
Workflow lebih sederhana
Kesimpulan: Lebih efisien → GO

6. Service (Layanan)
User experience lebih baik (UI, akses cepat)
Admin lebih mudah mengelola website
Kesimpulan: Layanan meningkat → GO
