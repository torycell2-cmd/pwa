1. Estimasi Ukuran & Biaya Software
LOC (Lines of Code)
Mengukur ukuran software dari jumlah baris kode
Semakin banyak LOC → semakin besar dan kompleks sistem
Kekurangan: tergantung bahasa pemrograman

Function Point (FP)
Mengukur berdasarkan fungsi sistem (bukan kode)
Fokus ke:
Input
Output
File
Interface
Lebih akurat dibanding LOC untuk perencanaan awal

COCOMO Model
Model untuk estimasi biaya & waktu pengembangan
Berdasarkan ukuran software (biasanya LOC)
Output:
Estimasi waktu
Jumlah tenaga kerja
Biaya

2. Elemen Utama RPL
PEOPLE
Semua orang yang terlibat:
Developer
Analyst
Project Manager
User

PRODUCT
Hasil akhir berupa software
Harus memenuhi kebutuhan user

PROSES
Langkah-langkah dalam pengembangan software

PROJECT
Pengelolaan keseluruhan:
Waktu
Biaya
Tim

3. Tahapan Pengembangan Software

PLANNING
Perencanaan proyek
Menentukan tujuan, biaya, waktu

ANALYSIS (Analising)
Analisis kebutuhan user
Apa saja fitur yang dibutuhkan

DESIGN
Perancangan sistem:
UI
Database
Arsitektur

IMPLEMENTATION
Coding / pembuatan program

4. Uji Dokumen Maintenance adalah proses:
Memeriksa, mengevaluasi, dan memperbarui dokumen sistem
Dilakukan saat software sudah digunakan dan masuk tahap maintenance
