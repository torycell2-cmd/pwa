ANALISIS FRAMEWORK PREP

penggunaan framework PREP (Point, Reason, Example, Point) bisa sangat efektif untuk menjelaskan apakah 
proyek Content Management System layak dikerjakan atau tidak (keputusan go / no-go).

Berikut penjelasan dengan metode PREP

1. Point (Pernyataan Utama)
Proyek CMS layak dikerjakan (GO) karena mampu meningkatkan efisiensi pengelolaan konten dan kualitas layanan sistem informasi.

2. Reason (Alasan)
Ada beberapa alasan utama:
CMS mempermudah pengelolaan konten tanpa perlu coding
Sistem lama biasanya tidak fleksibel dan sulit di-update
CMS meningkatkan efisiensi kerja admin dan tim
CMS modern lebih aman dan scalable
Artinya, dari sisi teknis dan bisnis, CMS memberikan banyak keuntungan dibanding sistem manual atau lama.

3. Example (Contoh Kasus)
Misalnya:
Sebuah perusahaan masih mengelola website secara manual (hard coding HTML)
Setiap update harus melalui programmer → memakan waktu lama
Sering terjadi kesalahan input dan keterlambatan informasi
Dengan CMS:
Admin bisa langsung update konten
Proses jadi lebih cepat
Informasi selalu up-to-date
Ini menunjukkan adanya peningkatan signifikan → mendukung keputusan GO

4. Point (Penegasan Ulang)
Dengan mempertimbangkan manfaat efisiensi, kemudahan penggunaan, dan peningkatan layanan, maka proyek CMS layak untuk dikerjakan (GO).

ANALISIS FRAMWORK PIECES

Kenapa proyek CMS layak dianalisis dengan PIECES?
Framework PIECES membantu mengevaluasi sistem dari 6 aspek utama:

1. Performance (Kinerja)
CMS sering digunakan banyak user (admin, editor, pengunjung).
Apakah website cepat diakses?
Apakah server kuat menangani traffic tinggi?
Kalau CMS lama lambat → indikasi perlu dikembangkan (GO)

2. Information (Informasi)
CMS berfungsi mengelola konten.
Apakah informasi yang ditampilkan akurat?
Apakah update konten mudah?
Jika informasi tidak up-to-date → GO

3. Economy (Ekonomi)
Menilai biaya vs manfaat.
Apakah biaya maintenance CMS lama tinggi?
Apakah sistem baru lebih hemat?
Jika sistem baru lebih efisien → GO

4. Control (Kontrol / Keamanan)
CMS rawan terhadap:
hacking
akses tidak sah
Jika keamanan lemah → GO (perlu perbaikan sistem)

5. Efficiency (Efisiensi)
Apakah admin mudah mengelola konten?
Apakah workflow ribet?
Jika proses kerja lambat → GO

6. Service (Layanan)
Apakah user puas?
Apakah tampilan user-friendly?
Jika UX buruk → GO
